## 1. Phase 3 mission

**Open the platform beyond agency-mediated work. Direct brands self-serve. Creators discover and apply to campaigns. AI starts driving real value. Multi-region.**

Phase 3 transforms Catalyst Engine from a managed-service operating system (Phases 1–2) into a true two-sided marketplace with intelligence.

---

## 2. Success criteria

- 5,000+ creators
- 50+ direct brands self-serving (not via agencies)
- 5+ agencies in production
- Defined GMV target (set during planning)
- SOC 2 Type 2 audit period began
- ISO 27001 gap analysis completed
- Multi-region (EU + LATAM at minimum)
- AI features live with measurable impact

---

## 3. Major features

### 3.1 Direct brand self-serve

- Brand sign-up flow without an agency
- Company verification (business email, website check, optional ID)
- Self-serve campaign creation with stronger guardrails
- SaaS pricing tiers: starter / growth / enterprise
- Stripe Billing for subscription + per-transaction fees

### 3.2 Creator marketplace (job board)

- Creators browse open campaigns and apply
- Personalized campaign feed based on AI matching
- Direct invitations from brands continue to work alongside applications
- Per-agency toggle: agencies can keep their roster in "internal-only" mode (Phase 1 model) or open creators to public marketplace
- Application + brief response flow
- Applicant ranking for brands

### 3.3 AI matching

- Recommendation engine based on past campaign performance, audience fit, content style embeddings
- Vector embeddings on creator content (Phase 3 introduces an embedding service)
- Semantic search across creators
- "Find creators similar to X" lookalike search
- Embeddings stored in pgvector or dedicated vector DB

### 3.4 AI brief generation

- Brand describes goal in natural language → AI drafts a structured brief
- Editable by brand before publishing
- Templates per industry / objective
- Multilingual (en, pt, it)

### 3.5 AI pricing recommendations

- "Creators in your niche typically charge $X for a Reel"
- Based on platform aggregate data
- Confidence intervals shown
- Helps brands set fair rates and creators set competitive rates

### 3.6 AI performance prediction

- Before campaign launch: predicted reach, engagement, ROI
- Trained on historical campaign data
- Confidence scores
- Shown as ranges, not point estimates

### 3.7 AI content QC

- Automated checks against brief specifications:
  - Required hashtags present
  - Required mentions present
  - Required links present
  - Camera framing assessment (vertical for Reels, etc.)
  - Audio quality assessment
  - Script adherence (if script provided)
  - Brand mentions present
- Runs before human review; reduces review cycles
- Pass/fail with detailed report
- Human review still required for final approval

### 3.8 Auto-generated reports

- Post-campaign reports generated by LLM from data
- Narrative + data + visualizations
- Customizable templates
- Exportable PDF for clients

### 3.9 Workflow automation builder

- Generic if-then-else rule builder for boards
- Trigger / condition / action picker
- More flexible than Phase 1's smart automation
- Per-tenant rules can be saved as templates
- Limited variable set; not full programming

### 3.10 Advanced scripting and review workflows

- Scripted approval chains (legal review → brand standards review → final approval)
- Multi-stage review with assignments
- Review SLAs

### 3.11 Multi-region deployment

- LATAM expansion (matching Influur's footprint), Brazil, Mexico
- Localized payment methods (Pix in Brazil, OXXO in Mexico, SEPA in EU)
- Multi-language UI: add Spanish, possibly French
- Per-region tax compliance (VAT, IVA, etc.)
- Regional pricing
- Regional creator pools

### 3.12 Anti-fraud and trust

- Anomaly detection ML for suspicious engagement patterns
- Anomaly detection for payment fraud
- Verified creator tiers (premium, top-tier)
- Tier-based access to higher-paying campaigns
- Creator reputation scoring
- Brand reputation scoring

### 3.13 Public API (read + write)

- Documented at `/api/docs` publicly
- OAuth2 (Laravel Passport)
- Scope-based permissions
- Rate-limited per token
- Webhook subscriptions outbound
- SDK in TypeScript and Python

### 3.14 Initial integrations exposed

- Shopify (track conversions from campaigns)
- Klaviyo (creator email lists)
- Google Analytics
- Meta Ads Manager (basic data sync)
- Zapier integration

### 3.15 Enterprise sales features

- Custom MSAs
- Procurement integration (security questionnaires automated)
- Dedicated CSMs
- Custom SLAs
- SAML SSO via Okta, Azure AD
- Audit log export to SIEM
- Custom data residency options

## 4. Database changes (Phase 3)

- Marketplace columns activated on campaigns (`is_marketplace_visible`, `marketplace_open_at`, `marketplace_close_at`)
- `creator.tier` column activated
- AI-related tables: `ai_models`, `ai_predictions`, `ai_embeddings` (or dedicated vector DB)
- AI QC columns activated on drafts (`ai_qc_results`, `ai_qc_passed`)
- Public API token expansion (scopes table)
- Marketplace application table: `campaign_applications`
- Subscription tier definitions and per-tenant overrides
- Multi-region tables: regional pricing, regional payment methods
- Reputation score tables
- Workflow rule builder tables: `workflow_rules`, `workflow_conditions`, `workflow_actions`
